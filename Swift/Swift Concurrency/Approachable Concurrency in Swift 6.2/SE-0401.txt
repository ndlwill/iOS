Remove Actor Isolation Inference caused by Property Wrappers
Status: Implemented (Swift 5.9)
Upcoming Feature Flag: DisableOutwardActorInference
Always enabled in Swift 6 language mode

#####
Starting in Swift 6, actor isolation must now be explicitly declared on the type, 
making isolation clearer and easier to understand. 
#####

@propertyWrapper
struct UIUpdating<Wrapped> {
@MainActor var wrappedValue: Wrapped
}

struct CounterView {
@UIUpdating var intValue: Int = 0
}
之前 (旧行为)：CounterView 会被 推断 (inferred) 为 @MainActor，因为它有一个 property wrapper，其 wrappedValue 是 @MainActor。
改动后 (提案生效)：CounterView 不再自动变成 @MainActor，除非你自己在类型上加 @MainActor。

在 Swift 6 语言模式下，完全移除这个由 property wrapper 引发的 actor 隔离推断。
也就是说：即使属性包装器的 wrappedValue 带有 actor，也 不会自动把整个类型标记为那个 actor。
