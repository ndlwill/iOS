为什么要存储数据?
在开发中展示的大部分数据是从网络服务器中加载出来的
但是如果所有的数据每次都从网络请求,则会有相应的问题.
浪费用户的流量
在用户没有网络时,无法正常显示页面

将网络数据存储在本地，不用每次都加载
减少用户网络流量开销
用户在没有网络时,依然可以展示一些数据

存储方式
Plist（NSArray\NSDictionary）
Preference（偏好设置\NSUserDefaults）
NSCoding（NSKeyedArchiver\NSkeyedUnarchiver)
SQLite
Core Data

在所有的存储方式中,SQLite速度最快,效率最高.

数据库可以分为2大种类(了解)
关系型数据库（主流)
对象型数据库

常见的关系数据库
PC端：Oracle、MySQL、SQL Server、Access、DB2
嵌入式\移动客户端：SQLite

关系数据库的特点
一个 字段(COL) 存储一个值，类似于对象的一个属性
一 行(ROW) 存储一条记录，类似于一个对象
一个 表(TABLE) 存储一系列数据，类似于对象数组
多个 表 之间存在一定 关系，类似于对象之间的关系，例如：一条微博数据中包含用户记录


SQLite的使用步骤
新建一个数据库->一个用于存储数据的文件
数据库中创建表->一个表中用于记录一系列数据
创建表时需要指定该表有哪些字段
比如学生表有:学号/姓名/年龄/身高等
在表中添加数据
之后对表进行增删改查操作
增:给表中添加数据
删:从表中删除数据
改:修改表中原有数据
查:查询表中原有数据

Navicat软件的介绍
Navicat是一款著名的数据库管理软件，支持大部分主流数据库（包括SQLite）


字段(Field/Col)：一个字段存储一个值，可以存储 INTEGER, REAL(浮点), TEXT, BLOB(二进制), NULL 五种类型的数据
SQLite 在存储时，本质上并不区分准确的数据类型
主键：Primary Key
为什么需要主键?
如果t_student表中就name和age两个字段，而且有些记录的name和age字段的值都一样时，那么就没法区分这些数据，造成数据库的记录不唯一，这样就不方便管理数据
良好的数据库编程规范应该要保证每条记录的唯一性，为此，增加了主键约束
也就是说，每张表都必须有一个主键，用来标识记录的唯一性

主键的设计原则
主键应当是对用户没有意义的
永远也不要更新主键
主键应当由计算机自动更新增长
主键不能为空
