常见的分组加密（block cipher）模式

| 模式      | 全称                         | 特点简介                    | 并行性          | 是否需要IV           | 常见用途              |
| ------- | -------------------------- | ----------------------- | ------------ | ---------------- | ----------------- |
| **ECB** | Electronic Codebook        | 每块独立加密；简单但不安全           | ✅ 可以并行       | ❌ 不需要            | 几乎不用              |
| **CBC** | Cipher Block Chaining      | 每块依赖上一个密文；安全性高但不能并行     | ❌ 加密不可并行；解密可 | ✅ 需要随机IV         | 文件、旧版TLS          |
| **CFB** | Cipher Feedback            | 类似流加密；可加密非整块数据          | ⚙️ 部分可并行     | ✅                | 流式通信              |
| **OFB** | Output Feedback            | 完全流加密；错误不传播             | ✅ 可并行        | ✅                | 实时通信、低延迟场景        |
| **CTR** | Counter                    | 用计数器生成伪随机流，与明文异或        | ✅ 完全可并行      | ✅（Nonce/Counter） | 现代常用模式之一          |
| **GCM** | Galois/Counter Mode        | CTR + 认证标签（MAC），同时加密+认证 | ✅ 完全可并行      | ✅（Nonce）         | HTTPS、VPN、TLS 1.3 |
| **XTS** | XEX-based Tweaked CodeBook | 为磁盘加密设计（支持随机访问）         | ✅ 可并行        | ✅                | 硬盘/文件系统           |
| **CCM** | Counter with CBC-MAC       | CTR + CBC-MAC，一种认证加密模式  | ✅            | ✅                | 无线通信（如AES-CCM）    |


按类型划分
1️⃣ 传统加密模式
这些模式只保证机密性（confidentiality），不保证数据完整性。
| 模式                  | 特点                            |
| ------------------- | ----------------------------- |
| **ECB**             | 每块独立加密，效率高但极度不安全。相同明文 → 相同密文。 |
| **CBC**             | 相邻块链式加密，解决了ECB问题。             |
| **CFB / OFB / CTR** | 把分组算法变成“流加密”，可以处理任意长度数据。      |

2️⃣ 认证加密模式（Authenticated Encryption, AE）
这些模式同时提供：
🔒 加密（confidentiality）
🧾 认证（integrity）
| 模式                            | 特点                                     |
| ----------------------------- | -------------------------------------- |
| **GCM (Galois/Counter Mode)** | 基于CTR，附带GHASH计算认证标签，现代标准（TLS、IPsec常用）。 |
| **CCM (Counter + CBC-MAC)**   | 用CTR加密 + CBC-MAC认证，适合嵌入式（如WiFi WPA2）。  |
| **EAX、OCB、SIV**               | 其他设计精巧的认证模式，部分在军用或研究领域使用。              |


| 特性      | ECB | CBC | CTR | GCM |
| ------- | --- | --- | --- | --- |
| 是否安全    | ❌   | ✅   | ✅   | ✅✅  |
| 是否能并行加密 | ✅   | ❌   | ✅   | ✅   |
| 是否能并行解密 | ✅   | ✅   | ✅   | ✅   |
| 是否可随机访问 | ✅   | ❌   | ✅   | ✅   |
| 是否带认证   | ❌   | ❌   | ❌   | ✅   |


| 需求            | 推荐模式       |
| ------------- | ---------- |
| 需要最高安全性（现代标准） | 🔹 **GCM** |
| 只做加密，追求速度     | 🔹 **CTR** |
| 旧系统兼容性        | 🔹 **CBC** |
| 文件/磁盘加密       | 🔹 **XTS** |
| 嵌入式/WiFi      | 🔹 **CCM** |
