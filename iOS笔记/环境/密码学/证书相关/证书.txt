“证书”就是“公钥证书”，也叫 数字证书 或 X.509 证书（这是最常见的格式）。

公钥证书（Public Key Certificate） = 一个绑定了 公钥 和 身份信息 的文档，并由一个 受信任的证书颁发机构（CA） 对其内容进行签名。



==================================================
证书（Certificate）、私钥（Private Key）和公钥（Public Key）
证书 = 公钥 + 身份信息 + 签名，而私钥是保密的配对密钥，用于加密或签名。

用公钥加密 → 私钥解密（用于数据保密）
用私钥加密（签名） → 公钥验证（用于身份认证）


证书是由**证书颁发机构（CA）**签发的一个文件，主要结构是：
┌────────────────────────────┐
│        证书内容            │
│                            │
│  - 版本号（Version）       │
│  - 序列号（Serial Number） │
│  - 签名算法（Signature Alg）│
│  - 发行者信息（Issuer）     │
│  - 有效期（Validity）       │
│  - 持有者信息（Subject）    │
│  - 公钥（Subject Public Key Info）│
│                            │
└────────────────────────────┘
           ↓
┌────────────────────────────┐
│   使用 CA 私钥对上面部分签名 │
└────────────────────────────┘


身份信息（Identity Info）包括哪些？
这是证书的主体内容，表示“谁”的公钥，这些信息包括：
1. Subject（证书持有者的信息）
CN（Common Name）：如域名 www.example.com 或人名
O（Organization）：组织名
OU（Organizational Unit）：部门名
C（Country）：国家，如 CN、US
ST（State）：省/州
L（Locality）：城市

2. Issuer（签发者的信息）
跟 Subject 类似，但表示“谁签发了这张证书”，一般是 CA。

3. Validity（有效期）
Not Before：证书生效时间
Not After：证书过期时间

4. Subject Public Key Info（公钥信息）
公钥算法类型（如 RSA、ECDSA）
公钥数据本体


签名（Signature）是什么？
这是 CA 用它的私钥对证书中「所有字段（不包含签名本身）」进行的数字签名，用于确保：
证书的内容未被篡改
证书确实是由某个受信 CA 签发的
签名包括：
签名算法（如 sha256WithRSAEncryption）
签名值（即加密过的哈希值）
验证时，浏览器用 CA 的公钥 来验证这个签名是否对得上 → 说明证书可信。


身份信息 是描述证书“属于谁”的详细身份 + 有效期等元数据；
签名 是由 CA 用私钥对这些信息 + 公钥生成的指纹，确保信息完整和来源可信。
