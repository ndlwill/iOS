Window Scale 是一个 TCP 可选项（TCP Option），它会被附加在 TCP 头部后面，从而让 TCP 头部长度变大。

标准的 TCP 头部（不带任何选项）长度是固定的
20 字节（5 × 4 字节，Header Length = 5）

TCP 为了扩展功能（比如窗口缩放、时间戳、SACK 等），在头部末尾预留了一个“可选字段”区域，最大可扩展到：
TCP Header 最大长度 = 60 字节
也就是说，TCP Header 最多可以包含 40 字节的 Options。

TCP Header 增长:
这会影响：
MSS（最大分段大小）：数据部分必须减少，避免超过 MTU。
性能分析和抓包调试：抓包时看到 TCP 头部长度 > 20，一般就说明启用了某些 TCP 扩展。


==================================================Window Scale
Window Scale 本身不是“可接受的数据大小”，而是一个 缩放因子。
但它间接决定了 TCP 的“接收窗口大小”（也就是一次能接受的数据量）。

什么是接收窗口（Receive Window）？
TCP 的 接收窗口表示：“我这个接收方，当前最多还能接受多少字节的数据”。
发送方会参考这个值，决定自己能发多少数据不需要等 ACK。

TCP 头里的接收窗口字段只有 16 位，最大就是：
2^16 - 1 = 65535 字节（64KB）
这个在现代网络里太小了，所以引入了 Window Scale（窗口缩放） —— 让这个值可以放大
实际窗口大小 = 窗口字段的值 × (2^WindowScale)
