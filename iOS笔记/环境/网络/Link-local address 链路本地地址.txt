“Link-local address”（链路本地地址）是一种在网络中仅在同一链路（同一网络段）上有效的 IP 地址。
它不能被路由到其他网络，也就是说，只能在局域网或者同一广播域内通信。

IPv4 的 Link-local 地址
IPv4 link-local 地址范围：169.254.0.0/16（也就是 169.254.0.0 ~ 169.254.255.255）。
用途：
当一个设备无法通过 DHCP 获取到有效的 IP 地址时，会自动分配一个 link-local 地址，这样局域网内的设备仍然可以相互通信。
这种机制也叫 APIPA（Automatic Private IP Addressing）。
特点：
不需要手动配置，也不依赖 DHCP。
仅在同一网络段有效，不能跨路由器通信。
示例：
192.168.1.10  # 普通局域网IP
169.254.10.23 # link-local


IPv6 的 Link-local 地址
IPv6 link-local 地址范围：fe80::/10（也就是以 fe80 开头的地址）。
每个 IPv6 网络接口都必须有一个 link-local 地址，即使没有全局地址。
用途：
节点之间在本地链路上的通信（如路由器邻居发现、自动配置等）。
IPv6 路由器之间的通信也经常依赖 link-local 地址。
特点：
必须与接口绑定（比如 fe80::1%eth0，%eth0 表示绑定到网卡）。
不可路由到其他链路。
示例：
fe80::1ff:fe23:4567:890a%eth0


link-local address 就是“只能在本地网段用的临时/必备 IP 地址”，不需要外部网络就能通信。



#####
IPv4 的场景（169.254.x.x）
自动分配 IP（APIPA）:
当设备无法从 DHCP 服务器获取到 IP 时，会自动分配一个 169.254.x.x 地址。
作用：
保证同一局域网内的设备仍然可以通信。
举例：两台电脑通过交换机直连，但没有 DHCP 或手动配置 IP，它们仍能互相 ping 通。

临时或测试网络:
在没有路由器或互联网的情况下，可以用 link-local 地址快速建立局域网通信。
用于小型临时网络或调试网络连接。
