As per Apple’s definition, 
“An Xcode toolchain (.xctoolchain) includes a copy of the compiler, lldb, and other related tools needed to provide a cohesive development experience for working in a specific version of Swift.”

Xcode 中的 Toolchain（工具链）用于指定 使用哪一套编译工具（如编译器、链接器、标准库等） 来构建你的代码。
Toolchain 是一整套编译相关的工具集合，通常包括：
编译器（比如 clang, swiftc）
标准库（如 libc++, Swift standard library）
链接器（ld）
头文件和模块（Headers, Modules）
可能还包括调试器等（lldb）

比如你安装了 Swift 5.9 的开发工具链，就可以选择用 Swift 5.9 来编译你的项目，而不是 Xcode 默认带的 Swift 版本。
能够轻松地测试最新的语言特性
Toolchain 决定你用哪个版本的 Swift 编译器来编译代码
SDK 仍然来自你当前 Xcode 自带的 iOS 17.5 SDK

SDK 版本确实是跟随 Xcode 版本的
SDK 实际上是位于 Xcode 的内部目录中的.SDK 一般不单独安装，是打包在 Xcode.app 内部的。Xcode 会自动使用其内置的 SDK
/Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/SDKs/iPhoneOS17.5.sdk
在 Xcode 项目的 build settings 中修改 SDKROOT
使用 xcodebuild 命令时通过 -sdk 参数手动指定
