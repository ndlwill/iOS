POSIX 是 Portable Operating System Interface 的缩写，中文一般翻作 可移植操作系统接口。

POSIX（Portable Operating System Interface）是一个针对类 Unix 系统的标准接口规范，定义了操作系统提供给程序的接口，包括 系统调用、库函数、工具命令 等。
它不是具体库，而是一套标准，符合 POSIX 的系统会提供对应的实现。

POSIX 是一套 操作系统接口标准（由 IEEE 制定）。
#####
目的是让不同的类 Unix 系统（Linux、macOS、BSD系统 等）在 API 层面保持兼容，这样写出来的程序更容易跨平台移植。
#####

常见实现：Linux 的 glibc、macOS 的 libSystem、BSD 系统的 libc。

它规定了：
文件操作接口（open, read, write, close）
进程控制（fork, exec）
线程（pthread_* 系列）
信号（kill, signal）
错误码（errno 里的那些）

在 macOS / iOS 里，很多系统 API（尤其是网络、文件、线程相关）出错时，会返回 POSIX 定义的错误码。

POSIX = 操作系统接口标准
POSIX error = 遵循该标准的错误码

Windows 和 POSIX 的差异:
POSIX 系统（Linux/macOS）:
线程库是 pthreads，mbedTLS 可以直接使用 pthread_mutex_t，库里有默认实现。
Windows:
Windows 没有 pthreads。
Windows 有自己的线程/锁 API，比如：
    CRITICAL_SECTION → 用户态锁，效率高
    Mutex → 内核态锁
mbedTLS 默认不会调用 Windows API，所以 必须提供自己的实现。

==================================================BSD = Berkeley Software Distribution
起源：1970s-1980s，美国加州大学伯克利分校对 Unix 系统的扩展版本。

常见 BSD 系统：
FreeBSD → 高性能服务器、网络应用
OpenBSD → 安全为主
NetBSD → 高可移植性，跑很多平台
DragonFlyBSD → 继承 FreeBSD，注重并发和文件系统

BSD 与 Linux 的关系
Linux 不是 BSD 系统，Linux 内核是 Linus Torvalds 1991 年独立开发的
Linux 和 BSD 都受 Unix 影响，很多系统调用、标准库、工具都是 POSIX 风格
Linux 是 Unix 风格的开源操作系统，BSD 是 Unix 的一个衍生家族

macOS 和 BSD 的关系
macOS（原来是 NeXTSTEP + Darwin）底层 Darwin 内核：
    Darwin 内核基于 Mach 微内核 + FreeBSD 的 BSD 内核子系统
    因此 macOS 提供了 POSIX API，很多命令行工具类似 BSD
这也是为什么你在 macOS 上可以用：
ls, grep, ps 等 BSD 命令
POSIX 线程（pthread）
mbedTLS 在 macOS 上可以直接使用 pthread 互斥锁


BSD = Unix 的一个衍生系统家族（FreeBSD、OpenBSD、NetBSD 等）
Linux 受 Unix/BSD 影响，但独立开发，内核+用户空间分离
macOS 底层用 BSD 内核子系统 + Mach 内核，所以它和 BSD 很接近


BSD 和 POSIX 的关系
BSD 系统实现了很多 Unix 功能，它是 POSIX 标准的早期实现参考
早期 POSIX 标准就是基于 BSD 和 System V 的接口制定的
现代 BSD 系统（FreeBSD、OpenBSD、NetBSD）都 符合 POSIX 标准
提供线程、文件 I/O、信号、时间、进程等 POSIX API
因此：
你在 BSD 系统上可以直接使用 POSIX 接口
POSIX 不是 BSD 独有，Linux、macOS 也都支持 POSIX


BSD 是具体系统，BSD 是 Unix 的一个衍生系统，POSIX 是标准规范。
BSD 系统实现了 POSIX，所以 BSD、Linux、macOS 都可以使用 POSIX API
