scale 和 nativeScale 是 UIScreen 的两个属性

scale: // The natural scale factor associated with the screen.
/*
This value reflects the scale factor needed to convert from the default logical coordinate space into the device coordinate space of this screen. 
The default logical coordinate space is measured using points. 
For Retina displays, the scale factor may be 3.0 or 2.0 and one point can represented by nine or four pixels, respectively. 
For standard-resolution displays, the scale factor is 1.0 and one point equals one pixel.
*/
定义：UIKit 的缩放因子。
含义：UIKit 将逻辑坐标（points）映射到物理像素（pixels）的比例。
例子：
普通屏幕（非 Retina）：scale = 1.0，1 point = 1 pixel
Retina @2x：scale = 2.0，1 point = 2×2 pixels
Retina HD @3x：scale = 3.0，1 point = 3×3 pixels
应用场景：
布局、绘制 UI 的时候都用这个值。
比如在 Core Graphics 绘图时，你可能会用 UIScreen.main.scale 来决定 CGContext 的 backing store 分辨率。


nativeScale: // The native scale factor for the physical screen.
定义：屏幕的 真实物理缩放因子。
含义：设备实际支持的物理像素与点的比例，可能和 scale 不同。
例子：
在 iPhone 6 Plus / 7 Plus / 8 Plus 上，UIKit scale 是 3.0，但 nativeScale 实际是 2.6087...（因为它们内部先以 @3x 渲染，再缩放到 1080p 屏幕）。
iPad Pro 上可能允许通过设置显示模式（放大显示/标准显示）让 nativeScale ≠ scale。


| 属性            | 含义                            | 用途             | 典型值              |
| ------------- | ----------------------------- | -------------- | ---------------- |
| `scale`       | UIKit 逻辑缩放因子（points → pixels） | UI 布局、绘图、图片加载  | 1.0, 2.0, 3.0    |
| `nativeScale` | 屏幕物理缩放因子（真实像素比率）              | 判断设备真实分辨率、渲染优化 | 可能是非整数，比如 2.6087 |
