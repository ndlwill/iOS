HTTP CONNECT 方法其实是 HTTP 协议里比较特殊的一种请求方法，它和普通的 GET、POST 不太一样，主要用在 代理服务器（proxy）场景下，尤其是 HTTPS 隧道（tunneling）。

CONNECT 的作用是告诉 HTTP 代理服务器：
    “我想和目标服务器建立一个 TCP 连接，请你帮我转发数据，不要处理 HTTP 内容。”
它通常用在访问 HTTPS 网站 或者其他需要加密通道的服务时。


请求示例:
假设你想通过代理访问 https://example.com:443，你的客户端会向代理发送：
CONNECT example.com:443 HTTP/1.1
Host: example.com:443
这里不是直接请求网页，而是请求代理建立到 example.com:443 的 TCP 连接。
代理如果同意，会返回：
HTTP/1.1 200 Connection Established
此时，客户端和目标服务器之间的通信就像 直接 TCP 连接，代理只负责 转发数据，不看内容。


为什么用 CONNECT？
HTTPS 是加密的，普通代理无法直接查看内容。
客户端需要 建立安全通道：
    使用 CONNECT 后，客户端可以直接对目标服务器执行 TLS 握手。
    代理只是“透明传输”，不会解密 HTTPS 内容。

CONNECT = 建立代理隧道，主要用于 HTTPS。
代理收到 CONNECT 后，不处理 HTTP，而是转发 TCP 数据。
用普通浏览器访问 HTTPS 网站时，如果配置了 HTTP 代理，浏览器背后就会用 CONNECT。
